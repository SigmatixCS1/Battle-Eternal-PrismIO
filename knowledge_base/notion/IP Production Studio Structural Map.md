# IP Production Studio Structural Map

Here is a clean framework you can use as the master outline before any creative writing or prompting.

High level dashboard for planning, not writing. Use each layer as a checklist of documents you maintain in the project folder before doing creative work.

#### 1. The Property: Battle Eternal

Treatment / Story Board

#### 2. 6-10 Series

Treatment / Story Board

#### 3. Each Series

Treatment / Story Board

#### 4. Each Episode

Treatment / Story Board / Script

### 1. IP Spine Layer

Defines the permanent “source of truth” documents.

IP Bible

Universe overview

Timeline & ages

Factions & power systems

Core themes & tone

Character Bible

Major characters (VN, TCG, mobile)

Archetypes & psychology

Long‑arc plans (who can change, who is fixed)

Transmedia Map

VN seasons & episodes

TCG product arcs

Mobile experiences

Future media (anime, books)

ICP & Brand Guardrails

Mythology Seeker ICP

Non‑negotiables (completion, mythology depth, community)

Brand voice & visual notes

### 2. Series Layer (6–10 Series Plan)

Each Series is a major multi‑season arc in the Battle Eternal IP.

For each Series:

Series Metadata

Title & code name

Medium focus (VN core, TCG heavy, mobile, mixed)

Target audience segment & ICP emphasis

Series Blueprint

Premise & stakes

Position on global timeline

Relationship to other Series (prequel, parallel, sequel)

Series Season Grid

Planned seasons for this Series

High‑level function of each season (intro, escalation, payoff)

Series Production Notes

Priority level

Dependencies on other Series or products

### 3. Season Layer (VN “Season” / Arc Layer)

Within one Series, define each Season as a contained story arc.

For each Season:

Season Metadata

Season label (S1, S2, etc.)

Episodes included (E1–E6 etc.)

Approximate total hours & asset scale

Season Concept Sheet

Core conflict & emotional throughline

Key theme focus (e.g. faith, betrayal, identity)

Season Structure Map

Opening state

Midpoint twist

Climax & resolution type (closed, cliffhanger, bittersweet)

Season Integration Notes

Where this plugs into transmedia (TCG hooks, mobile hooks)

Player community hooks (theory‑crafting questions, mysteries)

Season Production Trackers

Status fields (idea, outlined, scripting, in‑engine, shipped)

### 4. Episode Layer (Per Episode Framework)

This is the repeatable unit for VN production.

For each Episode:

Episode Metadata

Title & short logline

Length target (word count, playtime)

Production status

Episode Design Sheet

Narrative goal for the episode

Player promise (what they “get” by the end)

Primary locations & new characters introduced

Episode Structure Map

Beat list (in order)

Choice points (branch IDs, impact size)

Ending variants (flags only, no writing)

Episode Storyboard Shell

Scene ID

Scene type (dialogue, reveal, confrontation, downtime)

Visual/audio intent tags

Episode Tech Sheet

Required assets (BGs, sprites, animations, SFX, music)

Engine integration notes & complexity tags

### 5. Script & Prompting Layer

Defines how you will later write and use AI.

Script Format Spec

VN script markup standard

Tags for routes, choices, and conditions

VO, SFX, and animation call standards

Prompting Playbook

Standard prompt patterns for:

Dialogue drafting

Lore expansion

Scene ideation

Guardrails to protect canon and ICP

Review & Revision Flow

Draft → human edit → lore check → ICP check → final

Feedback tags for iteration (tone, pacing, clarity, canon)

### 6. Studio & Roadmap Layer

Connects IP planning to actual production.

Product Roadmap Grid

Rows: Series / Seasons / Episodes

Columns: Status, owner, target date, risk level

Metrics & Gates

Per Season and Episode success metrics

Clear gates to unlock next Season or Series

Cross‑Team Views

Narrative board

Art / animation board

Tech / engine board

Marketing / community board
